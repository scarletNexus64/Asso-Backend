ÉTAPES RAPIDES (POUR DÉVELOPPEURS)
================================================================================

1. Créer app/Http/Controllers/Api/ShopController.php
   → Copier le code complet de IMPLEMENTATION_GUIDE.md

2. Ajouter routes dans routes/api.php:
   Route::get('/shop', [ShopController::class, 'show']);
   Route::put('/shop', [ShopController::class, 'update']);
   Route::get('/shops', [ShopController::class, 'index']);

3. Créer migration:
   php artisan make:migration add_phone_email_to_shops_table
   → Copier contenu de IMPLEMENTATION_GUIDE.md

4. Exécuter migration:
   php artisan migrate

5. Mettre à jour Shop.php $fillable:
   Ajouter: 'phone', 'email'

6. Tester:
   curl -X GET http://localhost/api/v1/vendor/shop \
     -H "Authorization: Bearer TOKEN"

================================================================================
APIS EXISTANTES (FONCTIONNENT)
================================================================================

POST /api/v1/vendor/apply
- Crée une boutique
- Auth: Oui
- Retourne: Shop + User

GET /api/v1/vendor/dashboard
- Toutes les infos boutique + stats
- Auth: Oui
- Retourne: Shop + Stats + Package + Products

GET /api/v1/auth/profile
- Profil user + boutiques
- Auth: Oui
- Retourne: User + array de shops

================================================================================
APIS À CRÉER (PRIORITY 1)
================================================================================

GET /api/v1/vendor/shop
- Récupère infos complètes boutique
- Auth: Oui
- Retourne: Shop + Stats + Verification

PUT /api/v1/vendor/shop
- Modifie infos boutique
- Auth: Oui
- Paramètres: name, description, phone, logo, etc.
- Retourne: Shop mis à jour

GET /api/v1/vendor/shops
- Liste toutes les boutiques du vendeur
- Auth: Oui
- Retourne: Array de shops

GET /api/v1/shops/{id}
- Infos publiques d'une boutique
- Auth: Non
- Retourne: Shop public (visible uniquement si active/verified)

================================================================================
POINTS IMPORTANTS
================================================================================

SÉCURITÉ:
- Tous les endpoints /vendor/* requièrent auth:sanctum
- Vérification que user est vendeur (hasRole check)
- Vérification de propriété (user_id == auth()->id())

VALIDATION:
- shop_name: required, string, max:255
- shop_phone: nullable, string, max:20
- shop_logo: nullable, image, max:2048

ERREURS:
- 403: User not vendor ou pas propriétaire de la boutique
- 404: Shop not found
- 422: Validation error
